Codex로 첫 웹사이트 만들기

가입부터 GitHub Pages 게시까지, 터미널 없이 끝내기

이 가이드는 코딩을 처음 해보는 사람이 Codex와 GitHub 웹페이지만 사용해서 간단한 웹사이트를 만들고 인터넷에 게시하는 과정을 설명합니다.

오늘 만들 것

오늘 만들 프로젝트는 나의 첫 프로필 링크 페이지입니다. 인스타그램 프로필 링크 페이지처럼, 이름과 소개글, 자주 쓰는 링크 버튼이 있는 간단한 페이지입니다.

구분 내용
프로젝트 이름 나의 첫 프로필 링크 페이지
사용 도구 Codex, GitHub 웹사이트
설치 필요 없음. 터미널, Git 명령어 사용 안 함
최종 결과 인터넷 주소로 접속 가능한 웹페이지

전체 흐름

  1. GitHub 가입
  2. 새 저장소 만들기
  3. Codex에게 HTML 코드 만들게 하기
  4. GitHub 웹페이지에서 index.html 파일 만들기
  5. Codex가 만든 코드 붙여넣기
  6. Commit changes 클릭
  7. GitHub Pages 켜기
  8. 공개 주소로 접속하기

1단계. GitHub 가입하기

1-1. GitHub 접속

브라우저에서 github.com에 접속합니다.

1-2. Sign up 클릭

오른쪽 위의 Sign up 버튼을 누릅니다.

1-3. 이메일, 비밀번호, 사용자 이름 입력

사용자 이름은 나중에 웹사이트 주소에 들어갑니다. 

예시:
사용자 이름: happycoder123

나중에 만들어질 주소:
https://happycoder123.github.io/my-first-page/

확인

로그인 후 GitHub 메인 화면이 보이면 성공입니다.

2단계. 새 저장소 만들기

저장소(repository)는 웹사이트 파일을 보관하는 공간입니다. 쉽게 말해 내 웹사이트 전용 폴더입니다.

2-1. New repository 클릭

GitHub 화면 오른쪽 위의 + 버튼을 누른 뒤 New repository를 클릭합니다.

2-2. Repository name 입력

저장소 이름에 아래처럼 입력합니다.

my-first-page

2-3. Public 선택

GitHub Pages로 무료 게시하려면 처음에는 Public을 선택하는 것이 가장 쉽습니다.

2-4. Add a README file 체크

Add a README file을 체크합니다. 이렇게 하면 저장소가 비어 있지 않게 만들어져서 초보자가 다루기 쉽습니다.

2-5. Create repository 클릭

초록색 Create repository 버튼을 클릭합니다.

확인

저장소 화면에 README.md 파일이 보이면 성공입니다.

3단계. Codex에게 웹페이지 코드 만들게 하기

이제 Codex에게 웹페이지 코드를 만들어달라고 요청합니다. 아래 문장을 그대로 복사해서 Codex에 넣습니다.

Codex 요청문
나는 코딩을 처음 해보는 초보자야.

HTML, CSS, JavaScript를 하나의 파일에 모두 넣어서
"나의 첫 프로필 링크 페이지"를 만들어줘.

조건:
1. 파일명은 index.html
2. 모바일에서도 보기 좋게 만들어줘
3. 이름, 한 줄 소개, 링크 버튼 3개를 넣어줘
4. 링크 버튼은 Google, GitHub, ChatGPT로 해줘
5. 버튼을 클릭하면 새 창으로 열리게 해줘
6. 코드 안에 초보자가 이해할 수 있는 주석을 달아줘
7. 전체 코드를 한 번에 복사할 수 있게 보여줘

중요

Codex가 여러 파일로 나누려고 하면 이렇게 다시 말합니다. 

아니, 초보자라서 파일 하나로 하고 싶어.
HTML 파일 하나 안에 CSS와 JavaScript까지 모두 넣어서 다시 만들어줘.

4단계. GitHub 웹에서 index.html 파일 만들기

4-1. 저장소 화면으로 이동

방금 만든 my-first-page 저장소 화면으로 갑니다.

4-2. Add file 클릭

파일 목록 위쪽에 있는 Add file 버튼을 클릭합니다.

4-3. Create new file 클릭

메뉴에서 Create new file을 선택합니다.

4-4. 파일 이름 입력

파일 이름 입력칸에 아래처럼 입력합니다.

index.html

4-5. 코드 붙여넣기

큰 코드 입력창에 Codex가 만들어준 HTML 코드를 전부 붙여넣습니다.

5단계. 붙여넣을 예시 코드

Codex가 만든 코드가 마음에 들지 않거나 너무 복잡하면, 아래 코드를 그대로 사용해도 됩니다. 

<!doctype html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>나의 첫 프로필 링크 페이지</title>

  <style>
    /* 전체 화면 스타일 */
    body {
      margin: 0;
      font-family: sans-serif;
      background: #f3f4f6;
      color: #111827;
      text-align: center;
      padding: 40px 20px;
    }

    /* 가운데 카드 */
    .card {
      max-width: 420px;
      margin: 0 auto;
      background: white;
      border-radius: 24px;
      padding: 32px;
      box-shadow: 0 8px 24px rgba(0,0,0,0.1);
    }

    /* 프로필 원 */
    .profile {
      width: 90px;
      height: 90px;
      margin: 0 auto 20px;
      border-radius: 50%;
      background: linear-gradient(135deg, #2563eb, #7c3aed);
      color: white;
      display: flex;
      align-items: center;
      justify-content: center;
      font-size: 36px;
      font-weight: bold;
    }

    h1 {
      margin-bottom: 8px;
      color: #2563eb;
    }

    p {
      color: #4b5563;
    }

    /* 링크 버튼 */
    a {
      display: block;
      margin: 14px 0;
      padding: 14px;
      background: #2563eb;
      color: white;
      text-decoration: none;
      border-radius: 12px;
      font-weight: bold;
    }

    /* 마우스를 올렸을 때 */
    a:hover {
      background: #1d4ed8;
    }

    .small {
      margin-top: 24px;
      font-size: 13px;
      color: #9ca3af;
    }
  </style>
</head>

<body>

  <div class="card">

    <div class="profile">나</div>

    <h1>안녕하세요!</h1>

    <p>이것은 제가 처음 만든 웹페이지입니다.</p>

    <a href="https://www.google.com" target="_blank">Google 바로가기</a>

    <a href="https://github.com" target="_blank">GitHub 바로가기</a>

    <a href="https://chatgpt.com" target="_blank">ChatGPT 바로가기</a>

    <div class="small">
      Made with Codex
    </div>

  </div>

</body>
</html>

6단계. 파일 저장하기

6-1. Commit changes 클릭

화면 오른쪽 위 또는 아래쪽의 Commit changes 버튼을 클릭합니다.

6-2. 메시지는 그대로 두어도 됨

Commit message가 자동으로 들어가 있을 수 있습니다. 그대로 두거나 아래처럼 입력합니다. 

Add index.html

6-3. 다시 Commit changes 클릭

확인 창이 나오면 다시 Commit changes를 클릭합니다.

확인

파일 목록에 index.html이 보이면 성공입니다.

7단계. GitHub Pages 켜기

7-1. Settings 클릭

저장소 상단 메뉴에서 Settings를 클릭합니다.

7-2. Pages 클릭

왼쪽 메뉴에서 Pages를 클릭합니다.

7-3. Source 선택

Build and deployment 영역에서 Source를 Deploy from a branch로 선택합니다.

7-4. Branch 선택

Branch는 main, 폴더는 /root를 선택합니다.

7-5. Save 클릭

Save 버튼을 클릭합니다.

8단계. 내 웹사이트 접속하기

잠시 후 아래 형식의 주소로 접속할 수 있습니다. 

https://내깃허브아이디.github.io/my-first-page/

예를 들어 GitHub 아이디가 happycoder123이면 주소는 이렇게 됩니다. 

https://happycoder123.github.io/my-first-page/

바로 안 보이면?

GitHub Pages는 반영에 시간이 조금 걸릴 수 있습니다. 1~3분 정도 후 새로고침해 보세요.

9단계. 수정하는 방법

웹사이트를 수정하고 싶을 때도 Git 명령어는 필요 없습니다.

  1. GitHub 저장소로 이동
  2. index.html 클릭
  3. 오른쪽 위 연필 모양 Edit 클릭
  4. 내용 수정
  5. Commit changes 클릭
  6. 잠시 후 웹사이트에 반영됨

Codex에게 수정 요청 예시

아래 HTML 코드를 수정해줘.

변경하고 싶은 것:
1. 배경색을 밝은 노란색으로
2. 버튼을 5개로 늘리기
3. 내 소개 문장을 더 친근하게 바꾸기

전체 코드를 다시 보여줘.

10단계. 초보자가 자주 막히는 부분

문제 해결 방법
사이트가 404로 나옴 index.html 파일명이 정확한지 확인
주소가 안 열림 Pages 설정에서 main / root 선택했는지 확인
수정했는데 반영이 안 됨 1~3분 기다린 뒤 새로고침
코드가 깨짐 Codex에게 “전체 HTML을 다시 정리해줘”라고 요청
버튼 링크가 이상함 href="주소" 부분 확인

마지막 정리

이 과정을 한 번 끝내면 초보자는 아래 전체 흐름을 경험하게 됩니다.

  • Codex에게 원하는 웹페이지 설명하기
  • HTML 코드 받기
  • GitHub 저장소 만들기
  • GitHub 웹에서 파일 생성하기
  • GitHub Pages로 인터넷에 게시하기
  • 다시 수정하고 재게시하기

핵심

초보자는 Git 명령어를 먼저 배울 필요가 없습니다. 먼저 Codex로 만들고, GitHub 웹에서 올리고, 실제 주소로 확인하는 경험을 하는 것이 가장 중요합니다.
