초기 생성 SQL

로컬 PostgreSQL 또는 Supabase에서 처음 구축할 때 실행합니다.
주의: DROP TABLE은 기존 데이터를 완전히 삭제합니다. 운영 환경에서는 실행하지 마십시오.
postgresql 
-- =========================================================================
-- GH 재무 의사결정지원 시스템 · DB 초기 생성 스크립트
-- 현재 단계: master_project 단일 테이블
-- =========================================================================

DROP TABLE IF EXISTS master_project CASCADE;

CREATE TABLE master_project (
    year        INT          NOT NULL,
    half        VARCHAR(10)  NOT NULL,
    scenario    VARCHAR(50)  NOT NULL,
    description TEXT,
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    PRIMARY KEY (year, half, scenario),
    CONSTRAINT chk_year CHECK (year BETWEEN 2025 AND 2035),
    CONSTRAINT chk_half CHECK (half IN ('상반기', '하반기'))
);

COMMENT ON TABLE  master_project          IS '중장기 계획 마스터 프로젝트';
COMMENT ON COLUMN master_project.scenario IS '기본안 | 시나리오1~4 | 직접입력값';
Supabase 사용 시 추가 실행: 아래 RLS 블록을 SQL Editor에서 추가로 실행하세요.

postgresql · supabase only 
ALTER TABLE master_project ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "auth_all" ON master_project;
CREATE POLICY "auth_all" ON master_project
    FOR ALL TO authenticated USING (true) WITH CHECK (true);
MIGRATION NOTE FastAPI + 로컬 PostgreSQL 전환 시

FastAPI 전환 시 변경 사항

DB 스키마는 동일. 프론트엔드 js/config.js 두 줄만 수정하면 전환 완료.

현재 필요한 API 엔드포인트

FastAPI routes 
POST   /api/auth/login                                    → 로그인 (JWT 발급)
POST   /api/auth/logout                                   → 로그아웃

GET    /api/projects                                      → 전체 프로젝트 목록
POST   /api/projects                                      → 프로젝트 생성
DELETE /api/projects/{year}/{half}/{scenario}             → 프로젝트 삭제
PATCH  /api/projects/{year}/{half}/{scenario}/description → 설명 수정

프론트엔드 전환 방법

js/config.js 
DB_BACKEND:   'fastapi',
API_BASE_URL: 'http://localhost:8000/api',
