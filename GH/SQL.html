현행 테이블 구조

중장기 재무 의사결정 지원에 필요한 최소 구조입니다. 마스터 프로젝트 1개 + 하위 실적 테이블 4개로 구성되며, 복합 기본키(year, half, scenario)로 연결됩니다.

master_project

컬럼타입제약설명
yearINTPK, CHECK 2025~2030계획 기준 연도
halfVARCHAR(10)PK, CHECK 상반기|하반기반기 구분
scenarioVARCHAR(50)PK시나리오명 (기본안 · 시나리오1~4 · 직접입력)
descriptionTEXTNULLABLE시나리오 설명 (기본안은 NULL)

land_costs · development_costs · supplies · recoveries

4개 테이블 모두 동일 구조. master_project 복합키를 외래키로 참조 (ON DELETE/UPDATE CASCADE).
컬럼타입제약설명
idSERIALPK자동 증가 식별자
yearINTFK → master_project계획 기준 연도
halfVARCHAR(10)FK → master_project반기 구분
scenarioVARCHAR(50)FK → master_project시나리오명
detail_itemVARCHAR(200)NOT NULL세부 항목명 (엑셀 행 레이블)
target_yearINTNOT NULL실적/예산 귀속 연도
amountNUMERICDEFAULT 0금액 (원 단위)
인덱스 권장: 조회 성능을 위해 (year, half, scenario) 복합 인덱스를 각 하위 테이블에 추가하는 것을 권장합니다. 아래 초기 생성 SQL에 포함되어 있습니다.

전체 초기 생성 SQL

로컬 PostgreSQL 또는 Supabase에서 처음 구축할 때 실행하는 단일 스크립트입니다. 기존 테이블이 있으면 삭제 후 재생성합니다.
주의: 첫 번째 블록(DROP TABLE)은 기존 데이터를 완전히 삭제합니다. 운영 중인 환경에서는 실행하지 마십시오.
postgresql 
-- =========================================================================
-- GH 재무 의사결정지원 시스템 · DB 초기 생성 스크립트
-- 대상: PostgreSQL 14+ / Supabase
-- =========================================================================

-- 0. 기존 테이블 정리 (재구축 시 사용)
DROP TABLE IF EXISTS recoveries        CASCADE;
DROP TABLE IF EXISTS supplies          CASCADE;
DROP TABLE IF EXISTS development_costs CASCADE;
DROP TABLE IF EXISTS land_costs        CASCADE;
DROP TABLE IF EXISTS master_project    CASCADE;


-- =========================================================================
-- 1. 마스터 프로젝트 (master_project)
-- =========================================================================
CREATE TABLE master_project (
    year        INT          NOT NULL,
    half        VARCHAR(10)  NOT NULL,
    scenario    VARCHAR(50)  NOT NULL,
    description TEXT,
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    PRIMARY KEY (year, half, scenario),
    CONSTRAINT chk_year CHECK (year BETWEEN 2025 AND 2035),
    CONSTRAINT chk_half CHECK (half IN ('상반기', '하반기'))
);

COMMENT ON TABLE  master_project             IS '중장기 계획 마스터 프로젝트 (기준 연도·반기·시나리오)';
COMMENT ON COLUMN master_project.scenario    IS '기본안 | 시나리오1~4 | 직접입력값';
COMMENT ON COLUMN master_project.description IS '기본안은 NULL, 시나리오는 자유 기술';


-- =========================================================================
-- 2. 하위 4대 실적/예산 테이블 (공통 구조, CASCADE 외래키)
-- =========================================================================

-- 용지비
CREATE TABLE land_costs (
    id          SERIAL        PRIMARY KEY,
    year        INT           NOT NULL,
    half        VARCHAR(10)   NOT NULL,
    scenario    VARCHAR(50)   NOT NULL,
    detail_item VARCHAR(200)  NOT NULL,
    target_year INT           NOT NULL,
    amount      NUMERIC       NOT NULL DEFAULT 0,
    FOREIGN KEY (year, half, scenario)
        REFERENCES master_project(year, half, scenario)
        ON DELETE CASCADE ON UPDATE CASCADE
);

-- 조성비
CREATE TABLE development_costs (
    id          SERIAL        PRIMARY KEY,
    year        INT           NOT NULL,
    half        VARCHAR(10)   NOT NULL,
    scenario    VARCHAR(50)   NOT NULL,
    detail_item VARCHAR(200)  NOT NULL,
    target_year INT           NOT NULL,
    amount      NUMERIC       NOT NULL DEFAULT 0,
    FOREIGN KEY (year, half, scenario)
        REFERENCES master_project(year, half, scenario)
        ON DELETE CASCADE ON UPDATE CASCADE
);

-- 분양 수입
CREATE TABLE supplies (
    id          SERIAL        PRIMARY KEY,
    year        INT           NOT NULL,
    half        VARCHAR(10)   NOT NULL,
    scenario    VARCHAR(50)   NOT NULL,
    detail_item VARCHAR(200)  NOT NULL,
    target_year INT           NOT NULL,
    amount      NUMERIC       NOT NULL DEFAULT 0,
    FOREIGN KEY (year, half, scenario)
        REFERENCES master_project(year, half, scenario)
        ON DELETE CASCADE ON UPDATE CASCADE
);

-- 자금 회수
CREATE TABLE recoveries (
    id          SERIAL        PRIMARY KEY,
    year        INT           NOT NULL,
    half        VARCHAR(10)   NOT NULL,
    scenario    VARCHAR(50)   NOT NULL,
    detail_item VARCHAR(200)  NOT NULL,
    target_year INT           NOT NULL,
    amount      NUMERIC       NOT NULL DEFAULT 0,
    FOREIGN KEY (year, half, scenario)
        REFERENCES master_project(year, half, scenario)
        ON DELETE CASCADE ON UPDATE CASCADE
);


-- =========================================================================
-- 3. 인덱스 (조회 성능)
-- =========================================================================
CREATE INDEX idx_land_costs_proj        ON land_costs        (year, half, scenario);
CREATE INDEX idx_development_costs_proj ON development_costs (year, half, scenario);
CREATE INDEX idx_supplies_proj          ON supplies          (year, half, scenario);
CREATE INDEX idx_recoveries_proj        ON recoveries        (year, half, scenario);
Supabase 사용 시 추가 실행: 아래 RLS 블록을 SQL Editor에서 추가로 실행하세요 (로컬 PostgreSQL은 불필요).

Supabase 전용 — RLS 정책

postgresql · supabase only 
-- Supabase RLS: 로그인 세션(authenticated)만 모든 조작 허용
ALTER TABLE master_project    ENABLE ROW LEVEL SECURITY;
ALTER TABLE land_costs        ENABLE ROW LEVEL SECURITY;
ALTER TABLE development_costs ENABLE ROW LEVEL SECURITY;
ALTER TABLE supplies          ENABLE ROW LEVEL SECURITY;
ALTER TABLE recoveries        ENABLE ROW LEVEL SECURITY;

DO $$ DECLARE t TEXT;
BEGIN FOR t IN VALUES ('master_project'),('land_costs'),('development_costs'),('supplies'),('recoveries')
LOOP
    EXECUTE format('DROP POLICY IF EXISTS "auth_all" ON %I', t);
    EXECUTE format('CREATE POLICY "auth_all" ON %I FOR ALL TO authenticated USING (true) WITH CHECK (true)', t);
END LOOP; END $$;
MIGRATION NOTE FastAPI + 로컬 PostgreSQL 전환 시

FastAPI 전환 시 변경 사항

프론트엔드(js/db.js)와 백엔드(FastAPI) 연동에 필요한 API 엔드포인트 목록입니다. DB 스키마는 동일하게 유지됩니다.

필요한 API 엔드포인트

FastAPI route plan 
POST   /api/auth/login                                → 로그인 (JWT 발급)
POST   /api/auth/logout                               → 로그아웃

GET    /api/projects                                  → 전체 마스터 프로젝트 목록
POST   /api/projects                                  → 프로젝트 생성
DELETE /api/projects/{year}/{half}/{scenario}          → 프로젝트 삭제 (CASCADE)
PATCH  /api/projects/{year}/{half}/{scenario}/description → 설명 수정

GET    /api/tables/{table_name}                       → 전체 조회
GET    /api/tables/{table_name}?year=&half=&scenario= → 프로젝트별 조회
POST   /api/tables/{table_name}/bulk                  → 벌크 INSERT (엑셀 업로드)
DELETE /api/tables/{table_name}?year=&half=&scenario= → 프로젝트별 전체 삭제

프론트엔드 전환 방법

js/config.js 
// 이 두 줄만 수정하면 전환 완료
DB_BACKEND:   'fastapi',
API_BASE_URL: 'http://localhost:8000/api',
참고: 인증 방식은 Supabase JWT → FastAPI JWT(python-jose 등)로 교체합니다. js/db.js의 auth.signIn 구현만 수정하면 되며, 나머지 메서드 시그니처는 동일합니다.
