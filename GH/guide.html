사업별 재무 데이터를 의사결정에 바로 쓸 수 있게 정리합니다.

이 시스템은 엑셀 기반 원가·공급·회수 자료를 표준 데이터로 정리하고, 프로젝트별 재무 흐름을 대시보드, 조회 화면, 리포트로 확인하는 의사결정지원 도구입니다.

1. 좌측 메뉴

전체 요약, 업로드, 데이터 조회, 리포트를 업무 흐름 순서대로 배치합니다.

3. 본문 분석 영역

KPI 카드, 차트, 표가 같은 패널 규칙으로 정리되어 회의와 보고에서 바로 설명하기 쉽습니다.

4. DB 연동 상태

로그인 사용자와 DB 상태를 하단과 알림 영역에서 확인해 운영 신뢰도를 보여줍니다.

권장 업무 흐름
Standard Flow
01

인증 계정으로 로그인

Supabase Auth 세션으로 접근 권한을 확인한 뒤 시스템 화면으로 진입합니다.

02

프로젝트명 지정 후 엑셀 업로드

사업명 기준으로 원가, 공급, 회수 데이터를 DB 테이블에 적재합니다.

03

데이터 조회 및 리포트 생성

사업별 연도 흐름과 금액 단위를 선택해 피벗 조회, 차트 분석, 엑셀 내보내기를 수행합니다.

화면별 역할
Navigation
메뉴 설명

마스터 프로젝트 제어 등록된 사업 목록과 관리 작업을 확인하는 운영자 영역입니다.
데이터 세팅 / 업로드 엑셀 파일을 읽어 표준 DB 구조로 변환하고 저장합니다.
데이터 조회 원가, 공급, 회수 데이터를 사업 단위로 조회합니다.
현금흐름 리포트 연도별 투입·회수 흐름과 주요 재무 지표를 분석합니다.
데이터 운용 기준
Data Policy
  • 프로젝트명은 업로드, 조회, 리포트의 기준 키로 사용되므로 동일한 명칭 체계를 유지합니다.
  • 금액은 화면 상단의 출력 단위 선택값에 따라 천원, 백만원, 억원 단위로 환산됩니다.
  • 엑셀 원본의 연도 컬럼과 구분값은 DB 적재 후 차트와 피벗 구성에 사용됩니다.
  • 관리자 작업은 기존 프로젝트 데이터에 영향을 줄 수 있으므로 업로드 전 사업명을 확인합니다.
운영 화면에서 내보낸 엑셀은 현재 조회 조건과 금액 단위를 기준으로 생성됩니다.
사용 전 체크리스트
Ready Check
  • 로그인 계정이 Supabase 인증 사용자로 등록되어 있는지 확인합니다.
  • 업로드할 엑셀의 시트와 헤더가 최신 양식과 일치하는지 확인합니다.
  • 동일 프로젝트 재업로드 시 기존 데이터 처리 범위를 먼저 검토합니다.
  • 조회 결과가 비어 있으면 사업 선택값과 업로드 완료 상태를 확인합니다.

이 안내 페이지는 GH 공식 CI의 블루 계열과 의사결정지원 도구의 업무용 화면 구성을 함께 반영해 설명 자료로 활용할 수 있습니다.
